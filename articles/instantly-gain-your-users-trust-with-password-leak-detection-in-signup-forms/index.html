Introduction

tidak semua pengguna platform digital peduli pada jenis kebocoran data yang terjadi ada di internet. hal itu wajar karena tidak semua orang memiliki perhatian khusus pada web security, bahkan untuk seorang software engineer sekalipun.

tapi berbicara soal password adalah hal yang berbeda. tak peduli dia bodoh atau pintar mengenai teknologi, password akan selalu menjadi perhatian khusus bagi user. kenapa? karena ini lebih dari sekedar personal. dan skenario terburuk pada setiap orang adalah passwordnya diketahui orang lain. terlebih kecenderungan menggunakan hal yang sangat personal untuk password sudah sangat umum seperti mungkin nama ibunya, nama mantan, anime karakter, atau hal personal lain.

mekanisme password check detection melalui pustaka data breach

data breach telah terjadi lebih dari dekade yang lalu, dan terjadi pada perusahaan besar seperti adobe, twitter, facebook, gmail, canva, tokopedia. [source]

bersamaan dengan itu para developer juga sudah menambahkan beberapa fitur untuk meningkatkan keamanan entah itu menggunakan 2FA, OTP, security quiestions, MFA, hingga FIDO2. tak heran kini banyak perusahaan juga yang menawarkan security service bahkan khusus untuk menagani data breach seperti 1password, mozilla monitor, databreach.com, dan masih banyak lagi.

haveibeenpwned (HIBP)

sejauh yang saya ketahui, satu-satunya data pustaka yang memberikan access terbuka secara public untuk password breach hanya haveibeenpwned. (CMIIW), bahkan hampir semua security service terintegrasi dengannya.

haveibeenpwned adalah pustaka data breachs yang memberikan informasi kepada user tentang email, no telpon, password, dan hal lain yang telah bocor dan tersebar di internet.

haveibeenpwned dikembangkan oleh Troy Hunt. seorang pakar keamanan cyber yang saat ini menjavat sebagai microsoft regional director. troy hunt sejak lama mengumpulkan kata sandi dari berbagai sumber yang telah terekxpose di internet dengan tujuan membantu public mengetahui apakah kata sandi yang mereka gunakan sudah pernah terekspos sebelumnya, sehingga mereka dapat menghindari penggunaan kata sandi yang tidak aman.

hingga kini lebih dari 1 miliyar password breachs telah dikumpulkan dan telah terintegrasi di berbagai perusahaan dan security service seperti 1password, mozilla, github dan lain-lain.

saya percaya fitur password check detection potensial memberikan impact positif secara dalam bagi user experience dan psikologi pengguna.

mungkin beberapa orang akan terkejut saat mengetahui password yang ia gunakan ternyata pernah bocor/ pernah diketahui orang asing, dan bahkan iya telah gunakan bertahun-tahun. dan saat user mengganti passwordnya tentu dia akan merasa sangat terbantu, dan mungkin menggantinya juga di platform yang lain. dan disaat yang sama kita bisa mendapatkan impression positif. bukan hanya password barunya yang dia ingat. tapi juga product mu.

itu adalah expected behaviour, hal ini mungkin bisa berbeda apabila terjadi pada case lain.

namun jika fitur ini berjalan di platform financial industry(bank, payment gateway, blockchain) dan cloud service. kecenderungan mendapatkan positif feedback tidak akan jauh berbeda. terlebih jika kompetitor tidak memiliki ini. ini adalah pendapat pribadi saya dan sayangnya saya belum memiliki data yang ckup untuk mengukurnya.

Technical Implementation

kita akan menggunakan haveibeenpawned sebagai pustaka password breachs. dan sejauh ini kita dapat melakukan dengan 2 cara.

  1. menggunakan api https://api.pwnedpasswords.com , ini adalah bagian dari haveibeenpawned. troy hunt membuatnya bebas biaya, namun memiliki rate limit.

  2. self hosted haveibeenpawned password.
    kita perlu mendownload password breachs list dengan menggunakan haveipawned downloader, lalu mengembangkannya di server kita sendiri.

imolementasi pada frontend

untuk melindungi data privasi user, haveibeenpawned menggunakan K-Anatomy. setiap request yang kita lakukan, kita harus melakukan proses hash pada value yang akan kita kirim.

untuk tujuan umum, kita membuatnya di reactjs

import React from 'react';
import { useForm, SubmitHandler, FieldErrors } from 'react-hook-form';
import axios from 'axios';
import sha1 from 'sha1';

// Definisikan tipe untuk input form
interface IFormInput {
  password: string;
}

function PasswordCheckForm() {
  const { register, handleSubmit, setError, formState: { errors, isSubmitting } } = useForm<IFormInput>();

  // Fungsi untuk memvalidasi password menggunakan HIBP
  const checkPassword = async (password: string): Promise<boolean> => {
    // Hash kata sandi menggunakan SHA-1
    const hashedPassword = sha1(password);
    const prefix = hashedPassword.substring(0, 5); // 5 karakter pertama
    const suffix = hashedPassword.substring(5);    // Sisa karakter

    try {
      // Kirim prefix ke HIBP API
      const response = await axios.get(`https://api.pwnedpasswords.com/range/${prefix}`);
      const hashes = response.data.split('\n');

      // Cek apakah suffix ada dalam daftar hash yang dikembalikan
      const found = hashes.some(hash => {
        const [hashSuffix] = hash.split(':');
        return hashSuffix.toLowerCase() === suffix.toLowerCase();
      });

      return found;
    } catch (error) {
      console.error('Error checking password:', error);
      return false;
    }
  };

  // Handler untuk submit form
  const onSubmit: SubmitHandler<IFormInput> = async (data) => {
    const isPwned = await checkPassword(data.password);

    if (isPwned) {
      setError('password', {
        type: 'manual',
        message: 'Password telah ditemukan dalam pelanggaran data, gunakan kata sandi lain!',
      });
    } else {
      alert('Password aman!');
    }
  };

  const onError = (errors: FieldErrors<IFormInput>) => {
    console.error('Validation Errors:', errors);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit, onError)}>
      <div>
        <label htmlFor="password">Password:</label>
        <input
          type="password"
          id="password"
          {...register('password', { required: 'Password wajib diisi' })}
          aria-invalid={errors.password ? 'true' : 'false'}
        />
        {errors.password && <p style={{ color: 'red' }}>{errors.password.message}</p>}
      </div>

      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Checking...' : 'Submit'}
      </button>
    </form>
  );
}

export default PasswordCheckForm;
